# 0070 Documentation-Reality Alignment Process

## Status

Accepted

## Context

Phase 3 found README and ADR claims that were either stale or ahead of implementation.

## Decision

README feature claims must match tested controls. Limitations are listed explicitly. The real-data fixture runner and smoke test are the evidence for user-facing claims.

## Consequences

Docs become less flashy and more trustworthy. Future claims should land with tests.

## Alternatives Considered

Keeping aspirational language was rejected because the Phase 3 goal is stranger usability, not marketing.
